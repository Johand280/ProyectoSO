Contenedorización Modular con Docker Engine

Aislamiento de servicios frontend y backend compilados desde imágenes base limpias de Rocky Linux. Se orquestaron mediante Docker Compose acoplando un servidor web Nginx con un servidor de API escrito en Python a través de volúmenes compartidos y redes virtuales privadas de Docker.

Docker Engine Docker Compose Nginx Proxy Python HTTP API rockylinux:9-minimal
Servicio Imagen Base Utilizada Puerto Interno Responsabilidad
Frontend rockylinux:9 + nginx 80:80 Despliegue del index.html corporativo
Backend rockylinux:9 + python3 5000:5000 Procesamiento de peticiones del servidor
# docker-compose.yml estructurado
version: '3.8'
services:
  frontend:
    build: ./frontend
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
  backend:
    build: ./backend
    ports:
      - "5000:5000"
COMPONENTE 03 · 20%
Orquestación Elástica con Kubernetes & Minikube

Migración e inyección de la infraestructura elástica a nivel empresarial. Despliegue de un clúster local de Kubernetes mediante Minikube configurando un manifiesto YAML imperativo que define la replicación automatizada de los Pods y expone el tráfico de los microservicios de manera externa usando reglas de red NodePort.

Minikube Cluster Kubectl CLI K8s Deployment NodePort Service Horizontal Scaling
Elemento K8s Configuración Inicial Configuración Escalada Puerto NodePort
nginx-deployment 2 Réplicas (Pods Activos) 3 Réplicas (Manual Scale) Interno: 80
nginx-service Tipo: NodePort Balanceo Balanceado Externo: 30080
# deployment.yaml (Manifiesto de Orquestación)
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  type: NodePort
  selector:
    app: nginx
  ports:
    - port: 80
      targetPort: 80
      nodePort: 30080
// Comandos de Control y Verificación Ejecutados
# Lanzamiento y lectura del Manifiesto
kubectl apply -f deployment.yaml

# Comando estratégico para Escalado Manual a 3 réplicas
kubectl scale deployment nginx-deployment --replicas=3

# Inspección de salud de los nodos y puertos activos
kubectl get pods
kubectl get svc
